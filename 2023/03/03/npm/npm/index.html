发布到npm上

将终端切换到包的根目录，运行npm publish命令，即可将包发布到npm上（注意：包名不能雷同）

删除已发布的包

运行npm unpulish 包名 –force，即可从npm删除已发布的包。

补充：自定义模块的加载机制

加载自定义：如果省略了文件的扩展名，则Node.js会按顺序分别尝试加载以下的文件：

  1. 按照确切文件名进行加载
  2. 补全js扩展名进行加载
  3. 补全.json扩展名进行加载
  4. 补全.node扩展名进行加载
  5. 加载失败，终端报错。
    加载第三方模块：如果传递给require()模块标识不是一个内置模块，也没有以 “./“或 “../“开头，则Node.j会从当前模块父目录开始，尝试从/node_modules文件夹中加载第三方模块。
    如果没有找打对应第三方模块，则移动到再上一层父目录中，进行加载，直到文件系统的根目录。
    假如是：require(‘tools’) 查找数据如下
    C:User/itenia/project/node_modules/tools
    C:User/itenia/node_modules/tools
    C:User/node_modules/tools
    C:/node_modules/tools
